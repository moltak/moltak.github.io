## 개요

코딩 에이전트는 이제 동기형, 비동기형으로 나뉜다.

- **동기**: 커서, claude code, copilot 등 개발자가 응답이 나올때까지 기다려야하는 것
- **비동기**: cursor agent, jules, devin 처럼 브라우저에서 명령을 내리고 작업이 완료 된 후 슬랙, 이메일, PR, 브라우저 등으로 응답을 받는 것

## 소개

### Cursor Agent

- [https://cursor.com/agents](https://cursor.com/agents) - 유료 플랜 필요
- github repository 설정 후 브라우저에서 바로 명령을 실행
- 5분쯤 후 코드 수정 → 브라우저 알람

### Jules - Google 오픈베타 무료 (하루 60개 Task 생성 가능)

- [https://jules.google.com/](https://jules.google.com/) - 무료
- github repository 설정 필요
- 명령을 내리면 스스로 계획을 만들고 Approve를 물어봄 (5분 후 자동 승인)
- 5분쯤 지나니 새로운 제안과 함께 내 코드를 분석해줌
- 결과를 브랜치에 퍼블리싱 할지, PR 생성할지 물어봄
- 설정시 브라우저 알람도 전송됨

### Devin

- [https://app.devin.ai](https://app.devin.ai/) - 유료
- 명령을 내리면 스스로 판단하고 플랜을 짜고 실행
- 몇분 지나면 알람옴

## 결론

- 3개의 코딩 에이전트 모두 비동기로 동작함. 내가 개발환경을 세팅하거나 할 필요 없이 브라우저에서 모든 동작을 실행
- 제품에 따라 슬랙과 통합되는게 있어 슬랙에서 멘션으로 명령을 내리거나 결과 확인 가능하며 작업완료 됐을 때 노티도 받을 수 있음
- cursor의 경우 이메일 노티도 잘 되있고 메세지도 갈무리가 잘 되어있어서 보기가 편했음
